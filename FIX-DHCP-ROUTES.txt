FIX DHCP ROUTES - Type these one at a time

Step 1: Check what DHCP client is running
ps aux | grep dhcp

Step 2: Kill any DHCP processes (if found)
sudo pkill dhclient
sudo pkill dhcpcd

Step 3: Remove the DHCP default route
sudo ip route del default via 172.16.15.1 dev eth0 metric 100

Step 4: Remove other DHCP routes
sudo ip route del 1.1.1.1 via 172.16.15.1 dev eth0 metric 100
sudo ip route del 129.6.15.28 via 172.16.15.1 dev eth0 metric 100
sudo ip route del 132.163.96.1 via 172.16.15.1 dev eth0 metric 100

Step 5: Verify only ONE default route
ip route show | grep default

Should show only:
default via 172.16.15.1 dev eth0.15

Step 6: Test connectivity
ping -c 3 172.16.15.1

Step 7: If routes come back, check for DHCP client service
systemctl list-units | grep dhcp

Step 8: Disable any DHCP services found
sudo systemctl disable dhcpcd
sudo systemctl disable dhclient
